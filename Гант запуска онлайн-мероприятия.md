# Гант запуска онлайн-мероприятия на 7-10 дней

> Цель: быстро и структурно подготовить онлайн-мероприятие от идеи до проведения, рассылки материалов и анализа результата.

## Основная логика запуска

Запуск строится в четыре этапа:

1. **Проектирование** - тема, цель, формат, программа, спикеры.
2. **Упаковка** - лендинг, регистрация, визуалы, тексты.
3. **Продвижение и подготовка** - рассылки, соцсети, техническая настройка, репетиция.
4. **Проведение и follow-up** - эфир, запись, материалы, обратная связь, итоги.

## Ответственные

| Роль | Зона ответственности |
|---|---|
| Руководитель проекта | Общий контроль, сроки, команда, финальные решения |
| Продюсер | Программа, сценарий, спикеры, логика мероприятия |
| Маркетолог | Продвижение, регистрация, рассылки, рекламные каналы |
| Дизайнер | Баннеры, обложки, презентации, визуальный стиль |
| Технический специалист | Платформа, трансляция, запись, формы, доступы |
| Администратор | Коммуникация с участниками, напоминания, материалы |
| Спикеры / ведущий | Контент, выступление, удержание внимания аудитории |

## Контрольные точки

### День 1

- [ ] Выбрана тема мероприятия
- [ ] Понятна цель: регистрация, продажи, прогрев, обучение или комьюнити
- [ ] Определен формат: вебинар, мастер-класс, интенсив, эфир, разбор
- [ ] Назначен ответственный за запуск

### День 3

- [ ] Подтверждены спикеры
- [ ] Готова структура программы
- [ ] Есть черновик страницы регистрации
- [ ] Написаны базовые тексты для продвижения

### День 5

- [ ] Работает регистрация
- [ ] Готовы основные визуалы
- [ ] Настроена платформа эфира
- [ ] Проверены ссылки, формы и доступы

### День 7

- [ ] Проведена техническая репетиция
- [ ] Участникам отправлены напоминания
- [ ] Спикеры понимают сценарий и тайминг
- [ ] Есть план действий на случай технических проблем

### День 10

- [ ] Мероприятие проведено
- [ ] Запись сохранена
- [ ] Участникам отправлены материалы
- [ ] Собрана обратная связь
- [ ] Подведены итоги по регистрациям, посещаемости и результатам

## Минимальный набор материалов

- Страница регистрации
- Описание мероприятия
- Программа и тайминг
- Баннер или обложка
- Презентация спикера
- Ссылка на эфир
- Письма или сообщения с напоминаниями
- Форма обратной связи
- Запись мероприятия
- Follow-up сообщение после эфира

## Риски

| Риск | Что сделать заранее |
|---|---|
| Спикер не готов | Провести короткую репетицию и запросить тезисы заранее |
| Плохая явка | Сделать серию напоминаний за 24 часа, 3 часа и 30 минут |
| Не работает ссылка | Проверить доступы на нескольких устройствах |
| Нет записи | Назначить ответственного за запись и сделать тест |
| Слишком слабый оффер | Сформулировать понятную пользу для участника в одном предложении |

## Итоговый критерий готовности

Мероприятие готово к запуску, если участник может перейти по ссылке, понять пользу, зарегистрироваться, получить напоминание, зайти на эфир, посмотреть запись и получить материалы без ручного вмешательства команды.
